#### Technical Question Preparation (Week 7-8)

**Goal:** Answer technical questions confidently with homelab examples

**Common Technical Questions + Portfolio Answers:**

**Q: "What is defense in depth? Give an example."**
A: "Multiple layers of security controls. In my Raspberry Pi hardening, I implemented:
1. Network layer: Tailscale mesh VPN + LAN segmentation
2. Firewall layer: UFW with default-deny policy
3. Application layer: Vaultwarden restricted to Tailscale-only access
4. Intrusion prevention layer: fail2ban blocking brute-force attempts
5. Monitoring layer: Grafana alerting on security events
Each layer provides independent protection, so if one fails, others still defend."

**Q: "How does a SIEM work?"**
A: "My Home SOC uses SIEM principles:
1. Collection: Promtail agents ship logs from 7 devices to centralized Loki
2. Aggregation: Loki stores ~1000 log lines/minute with labels for filtering
3. Correlation: LogQL queries find patterns (SSH failures, container errors)
4. Visualization: Grafana dashboards show security events in real-time
5. Alerting: Triggered when thresholds exceeded (5 failed logins in 5 minutes)
I can detect SSH attacks fleet-wide and drill down to specific hosts using labels."

**Q: "Explain the vulnerability management lifecycle."**
A: "I follow a 6-step process in my homelab:
1. Discovery: Trivy scans all 30+ containers weekly
2. Prioritization: Risk-based scoring (CRITICAL/HIGH first, CVSS + exploitability)
3. Assessment: Determine applicability (is service exposed? compensating controls?)
4. Remediation: Patch if available, or apply compensating controls (Adminer example)
5. Verification: Rescan to confirm fix, test functionality
6. Documentation: Track in findings/ directory with CVE details and remediation steps"

**Q: "What is the difference between IDS and IPS?"**
A: "IDS detects and alerts (passive), IPS detects and blocks (active).
In my homelab, I use both:
- IDS: Grafana alerts on SSH failed logins (detection only, I decide response)
- IPS: fail2ban blocks IPs after 3 failed attempts (automatic prevention)
Planning to deploy Suricata for network IDS/IPS to detect malicious traffic patterns."
